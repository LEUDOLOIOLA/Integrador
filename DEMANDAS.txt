one hot

Agora vou aplicar todas as mudanças de uma vez: criar as colunas one-hot, atualizar FEATURES, e ajustar os dois pontos de montagem manual (simulação e inferência).

O que é One-Hot Encoding e por que usar aqui:

O DiaSemana original era um número de 0 a 6 (Seg–Dom). O problema é que o modelo interpreta isso como uma escala numérica, como se Domingo (6) fosse "maior" que Segunda (0) — o que não faz sentido.

O one-hot encoding cria 7 colunas binárias (0 ou 1):

Dia	DiaSemana_0	DiaSemana_1	DiaSemana_2	...	DiaSemana_6
Seg	1	0	0	0	0
Ter	0	1	0	0	0
Qua	0	0	1	0	0
Cada dia ativa apenas sua coluna (valor 1), todas as outras ficam 0. Assim o modelo trata cada dia como uma categoria independente, sem relação de ordem. Isso permite capturar padrões do tipo "às quartas o consumo é diferente" sem viés numérico.
